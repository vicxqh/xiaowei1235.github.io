    Github pages + jekyll的优势

    1. 管理方便 之前自己也有买过服务器，搭过wordpress，但是整体感觉就是折腾。尤其是当你买的服务器同时还在运行其它服务的时候，管理起来很麻烦.

    2. 写作更快捷 还有一个问就是用WordPress的话，需要花时间在布局上，而使用jekyll，采用markdown写博客则可以将注意力集中在内容本身。

    为何不适用最近很流行的hexo等其它SSG？

    通过hexo官方教程下载并体验了一天，整体感觉不如jekyll结构清晰。在定制化自己的网站时，我很清晰的知道，应该去哪里修改文件，以及怎么修改。

    建议只是push 生成的静态网页（_sitel下的内容)到github.io。 这样的好处是不用担心本地环境与github pages 所用环境不一致的问题。
