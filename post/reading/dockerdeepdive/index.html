总结

本书比较浅显，介绍了Docker的组成，并简单地介绍了工作原理。阅读次数，有助于理解Docker技术中的几个关键概念，比如Image、Container，Swarm, overlay network。

下一步计划阅读、学习kubernetes。

重要章节笔记

5. The Docker Engine

  • dockercli 作为用户交互的入口将dockerd的接口封装之后暴露给用户使用
  • dockerd 更像是一个网关服务，接受dockercli的请求并将其转发给后端containerd
  • containerd 负责管理容器的生命周期、以及镜像、网络等事务
  • runc作为独立的程序被containerd用于启动运行容器

6. Images

Image 是一个静态的描述文件，类似VM template（注意不是VM snapshot， Image并不能表达出内存所处的状态），而container则是从image创建而来的运行态的“应用容器”。

一个image由若干个只读层组成，位于上层的文件将隐藏下层的同名文件。

8 containerizing an app

DockerFile 描述了如何创建一个image。这些指令可以分为两类，一类会创建新的层，而另一类则仅仅是添加元数据。要注意如何编写DockerFile来优化镜像大小。

9 Swarm Mode

Docker的集群模式。参与到swarm中的节点被分为manager 和worker。manager 负责管理整个集群（docker的术语为swarm），具体管理哪些书本并没有细说，不过这个不重要。和其它系统一样，manager就是一个高可用的管理服务，而worker则是无状态的（？）的执行任务的工具人。

Swarm模式为了保证高可用，使用了etcd。

10 Docker overlay network

docker overlay network 是一个在docker节点之间基于应用层网络虚拟出来的2层vlan，使得位于不同物理节点上的container处于同一个虚拟的2层子网之中。如何在应用层实现vlan则依赖vxlan 技术。这种软件定义的网络（SDN）可以拥有更多的特性，比如通过加密来保证通信安全。
