协议特点

简单地对IP协议进行了一层包裹，添加了源、目的地端口，从而支持了多路复用、多路分解. 协议本身没有流控测流和拥塞控制策略。

好处

  1. 首部小， UDP 8字节， TCP20字节
  2. 应用层能更加精确地控制发送数据的时间

缺点，使用注意事项

  1. 不可靠。协议本身不保证应用层的数据，一定能完全达到目的地。但是每个UDP 包裹本身是数据是不会出错（UDP本身有校验和）
  2. 在网络上可能被很多节点的防火墙禁用掉

// Serve "监听"一个UDP端口，如"0.0.0.0:9527"
func (s *SimpleUDPServer) Serve(address string) error {
	c, err := net.ListenPacket("udp", address)
	if err != nil {
		return err
	}
	s.conn = c
	for {
        // 如果会使用到公网，UDP的包尽量不要太大, 500 字节内为宜
        // https://stackoverflow.com/questions/1098897/what-is-the-largest-safe-udp-packet-size-on-the-internet
        // 这里的maxBufferSize 应该 > 你设计的协议中最大包的大小
		buffer := make([]byte, maxBufferSize)
		n, addr, err := s.conn.ReadFrom(buffer)
		if err != nil {
			logs.Error("failed to receive udp packet: %v", err)
			continue
		}

        // 新启一个goroutine 来处理请求。主goroutine 不处理除了接受包之外的其它任何
		go s.handleRequest(buffer[:n], addr)
	}
	return nil
}
